Para que serve o commit e qual a melhor forma de utilizá-lo?

O commit no Git é essencial para registrar as mudanças feitas nos arquivos, assim criando um ponto na linha do tempo do projeto, permitindo acompanhar as alterações e voltar a versões anteriores se necessário. A melhor forma de utilizá-lo é dividir as mudanças em commits lógicos e significativos, cada um representando uma única funcionalidade ou correção isso facilita a revisão, o entendimento das alterações e a colaboração com outros membros da equipe.

Qual a diferença entre commit e push?

O commit registra as alterações localmente, enquanto o push envia esses commits para um repositório remoto como no GitHub. O commit é um passo local para salvar mudanças e enquanto o push sincroniza essas mudanças com o repositório online, assim permitindo que outros membros da equipe acessem e contribuam com o código.

O que é tag?

Uma tag no Git é uma referência estática a um commit específico que geralmente é usada para marcar versões importantes do código, como lançamentos de atualizações e são endereçadas com por exemplo (v1.0, v2.0). As tags ajudam a identificar e acessar facilmente versões específicas do código ao longo do tempo.

O que é branch?

Um branch é uma ramificação independente da linha principal do desenvolvimento (normalmente chamada de branch "master" ou "main"), permite que você trabalhe em novas funcionalidades, correções ou experimentos sem afetar diretamente o código na linha principal, as Branches são úteis para desenvolvimento paralelo e organização do trabalho em equipes.

Como resolver possíveis conflitos no Git?

Utilizando comandos como git status e git diff, é possível identificar os arquivos com conflitos, os conflitos devem ser resolvidos manualmente, editando os arquivos conflitantes para manter as mudanças relevantes. Após as edições, os arquivos devem ser adicionados (git add) e o commit realizado para concluir a resolução do conflito.
