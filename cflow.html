Com uso de Flow

Flow init:

Inicializa o repositório para usar o Flow, configurando o ambiente e as configurações necessárias para começar a trabalhar com o fluxo de trabalho definido.

Flow feature start :

Cria uma nova branch para desenvolver uma nova funcionalidade. Isso segue o conceito de criar branches separadas para cada funcionalidade a ser implementada.

Flow feature finish :

Conclui o desenvolvimento de uma funcionalidade específica e integra as mudanças de volta à branch principal (normalmente develop ou main), fechando a branch da funcionalidade.

Flow release start :

Cria uma nova branch para preparar uma versão de lançamento, permitindo correções finais e testes antes de lançar no ambiente de produção.

Flow release finish :

Conclui a versão de lançamento, integrando as alterações de volta à branch principal, adicionando uma tag para a versão e mesclando as alterações na develop ou main.

Flow hotfix start :

Inicia uma correção rápida em produção. Cria uma nova branch para corrigir bugs críticos que não podem esperar até a próxima versão planejada.

Flow hotfix finish :

Conclui a correção e aplica as mudanças tanto na branch principal quanto na develop ou main, assegurando que todas as versões tenham o bug corrigido.
